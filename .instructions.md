# HTML Cleaning Toolkit - Agent Development Instructions

## Quick Start for Agents

When continuing the development of this project, refer to this guide to understand the structure, architecture, and conventions.

### Project Overview

**Purpose:** HTML cleaning utility with three specialized tools for processing HTML bodies:

1. **Garbage Cleaner** - Remove scripts, styles, garbage attributes, and HTML entities
2. **Space Cleaner** - Remove unnecessary whitespace between tags (preserves inline element spacing)
3. **HREF Analyzer** - Interactively review and fix broken href references

**Current Status:** Full feature-complete toolkit with comprehensive test coverage (5/5 validation tests passing)

## Core Architecture

### Design Patterns Used

- **Repository Pattern**: All HTML operations encapsulated in module functions
- **Strategy Pattern**: Each cleaner is a specialized strategy (Garbage, Space, HREF)
- **Dependency Injection**: All dependencies passed via function parameters
- **Dataclass Pattern**: `CleaningResult` for immutable result tracking

### Module Responsibilities

#### garbage_cleaner.py

- **Main Function**: `clean_html(html: str) -> CleaningResult`
- **Exported**: `run_garbage_cleaner()` - CLI interface
- **Key Features**:
  - Removes `<script>` and `<style>` tags entirely
  - Filters attributes: keeps only `class` (all elements) and `href` (<a> tags)
  - Handles HTML entities properly with `convert_charrefs=False`
  - Removes `&nbsp;` entities (both `&nbsp;` strings and `\xa0` characters)
  - Returns statistics: scripts removed, attributes removed, entities removed, size reduction %

- **Entity Handling** (Critical):
  - Uses HTMLParser with `convert_charrefs=False` to intercept entities
  - Entity mapping dictionary converts Unicode back to HTML entities
  - Explicitly counts and removes `&nbsp;` from output

#### space_cleaner.py

- **Main Function**: `clean_spaces(html: str) -> tuple[str, int]`
- **Exported**: `run_space_cleaner()` - CLI interface
- **Key Features**:
  - Removes newlines between tags: `>\s*\n+\s*<`
  - Preserves spaces inside inline elements: `<span>`, `<a>`, `<strong>`, `<em>`, `<b>`, `<i>`, `<mark>`, `<small>`, `<code>`
  - Special pattern for anchors: `([a-zA-Z0-9])<a(\s|>)` prevents spacing issues

- **Important**: Space Cleaner must NOT touch spaces around text content - only between tags

#### href_analyzer.py

- **Main Function**: `run_href_analyzer()` - Interactive CLI only
- **Uses**: BeautifulSoup4 for HTML parsing (not HTMLParser)
- **Key Features**:
  - Detects incomplete hrefs (missing .htm/.html extensions)
  - Interactive prompts for user confirmation on each broken link
  - Modifies HTML in place based on user choices

### main.py Orchestrator

- Menu system with Rich CLI formatting
- Clipboard integration: `set_clipboard(text)` via PowerShell/clip command
- Import handling: Uses `sys.path.insert(0, 'modules')` for module loading
- Flow: Menu → Tool Selection → Input → Processing → Clipboard Output

- **Input**: User pastes HTML via Ctrl+V in CLI

### Result Format

All cleaners return or process HTML and provide statistics:


## Development Guidelines

### Code Quality Standards

1. **Single Responsibility**: Each function does exactly one thing
2. **Immutability**: Never mutate input parameters
3. **Error Handling**: Catch specific errors, never use empty `except` blocks
4. **Testing**: New features must include:
   - Unit tests in `tests/` folder
   - Validation in `tests/validate.py`
   - All test files prefixed with `test_`

### When Adding Features

1. **Module Expansion**: Add new functionality to appropriate existing module
2. **New Tool**: Create new file in `modules/`, export in `__init__.py`
3. **Testing**: Add test file with pattern `tests/test_<feature>.py`
4. **Validation**: Update `tests/validate.py` with new test case
5. **Documentation**: Update `docs/README.md` with new feature

## Common Issues & Solutions

### Issue: Import errors in tests

- **Cause**: Path not set correctly
- **Fix**: Ensure `tests/validate.py` has proper path setup:
  ```python
  sys.path.insert(0, parent_dir)
  sys.path.insert(0, os.path.join(parent_dir, 'modules'))
  ```

### Issue: Spaces being removed incorrectly

- **Cause**: Garbage cleaner might be manipulating whitespace
- **Fix**: Garbage cleaner should ONLY write raw data without whitespace changes
- **Verify**: Run `tests/test_space_preservation.py`

### Issue: HTML entities appearing as garbage characters

- **Cause**: HTMLParser auto-converting entities, then corrupting on output
- **Fix**: Use `HTMLParser(convert_charrefs=False)` and entity mapping dictionary
- **Example**: Map `'\xa0'` back to `'&nbsp;'` in output

### Issue: &nbsp; not being removed

- **Cause**: Entity not being intercepted
- **Fix**:
  1. Set `convert_charrefs=False` in HTMLParser
  2. Count entities in `handle_entityref('nbsp')`
  3. Remove both entity string `&nbsp;` and character `\xa0` in get_result()
